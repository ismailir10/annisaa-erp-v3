## Cross-cutting finding: header count race condition (minor)

Every list page renders the header entity count as "0" for ~2–3 seconds before the real value appears. Confirmed on: `/admin/payroll` ("0 riwayat penggajian"), `/admin` dashboard stat cards, and `/admin/students`. This is a client-side data fetch race — the initial render commits with a placeholder "0" that is visually indistinguishable from a real zero balance or empty list. On slower connections the window widens. Not a blocker, but Ibu Nur's stated priority is "see the cash position immediately" — a 3-second false-zero on the dashboard undermines this.
