### Pre-UAT incident — CRITICAL, not a product bug

Before the staging run began, a local seed attempt (`npx tsx prisma/seed.ts`) silently executed against the production Supabase pool because `.env` DATABASE_URL points at `aws-1-ap-southeast-1.pooler.supabase.com` and `.env.local` does not override. Seed runs ~20 `deleteMany()` calls at startup against the target DB. Pivoted to staging mid-run after detection.

**Remediation (recommended cycle):**
1. `prisma/seed.ts` — refuse any `DATABASE_URL` containing `supabase.com` unless `SEED_ALLOW_REMOTE=1` is set.
2. `.env.local` — provide a real local DATABASE_URL (sqlite file or local postgres), not empty supabase vars.
3. `README.md` setup — call out the dual-env hazard and the guard flag.

## Findings

### BLOCKER — PAY-03 BSI export click-to-visible 4148 ms

Payroll run → BSI export click → status flip "Slip Terkirim" → "Diekspor" took **4148 ms** measured from click to visible status change. Global `>3s click-to-visible = blocker` threshold breached. Per-job budget was `<3s for ≤50 employees`.

Additionally, file download was not visibly confirmed in-browser — Ibu Nur has no "file siap diunduh" toast or inline link.

**Why it matters:** payroll export is monthly, bank-upload-blocking, and under Friday-deadline pressure. 4s every click feels like the system hung, especially on mobile 4G where this will compound to 8-15s. A real user retries, triggering duplicate exports.

**Suggested fix:** background the export job, return optimistic status immediately, surface download URL via toast or poll-populated row action.

### BLOCKER — USER-01 no invite-user UI on /admin/settings/users

`/admin/settings/users` lists 33 users across 4 roles (Admin 2, Guru 28, Wali Murid 1, Tidak Aktif 0) with filters and sort — but there is **no "Tambah Pengguna", "Invite", or "Undang" button anywhere on the page**. Confirmed via DOM scan for any button/link matching `/tambah|invite|undang|buat pengguna|add/i` → zero matches.

JTBD USER-01 ("invite teacher user") has no UI path. Either users are auto-created at employee seeding (only works for initial load) or the invite flow is missing entirely. Admin adding a new teacher mid-year cannot grant login.

**Suggested fix:** add "Tambah Pengguna" primary button top-right of the page header, matching the pattern used on Karyawan/Hari Libur. Form fields: name, email, role, (optional) link-to-employee.

### BLOCKER (non-product) — Production DB wipe via seed

See Summary above. Add hostname guard + proper `.env.local` DATABASE_URL.

### MAJOR — INV-01 no manual single-invoice create

On `/admin/invoices`, the primary CTA "Buat Tagihan" opens the bulk monthly generator. There is no visible path for the JTBD INV-01 "weekly escape hatch: manually create single invoice for one student" — which the JTBD library explicitly calls out as a weekly-use workflow.

**Why it matters:** real admin workflow includes ad-hoc invoices (mid-month tuition adjustment, event fee for one family, seragam replacement). Forcing every invoice through bulk monthly flow means either (a) editing generated bulk runs (error-prone) or (b) ad-hoc not supported.

**Suggested fix:** row-action "Tambah tagihan manual" on Students list, or secondary CTA on /admin/invoices opening a single-student invoice form.

### MAJOR — EMP-02 create form does not capture salary components

`Tambah Karyawan` dialog fields: Nama, Nama Formal, Email, No. HP, Jabatan, Kampus, Tanggal Masuk, Bank, No. Rekening, BPJS Terdaftar. **No salary component / base salary input.**

JTBD EMP-02 expects "create employee + set salary" as one coherent task. Current flow is create → navigate to employee detail → set salary values separately. Two-trip workflow, easy to forget step 2, resulting in employee in payroll run with zero gaji.

**Suggested fix:** add optional "Gaji Pokok" + salary-component quick picker to create dialog, or surface an inline warning on employee detail page when SalaryValue rows are missing.

### MINOR — STUDENT-01 create form missing class + guardian inline

Create Student dialog captures: nama, nickname, gender, DOB, NIS, NISN, notes. JTBD expects class assignment and guardian capture as essentials — currently each is a separate page-level action after create. Three trips where one could suffice for common case.

**Suggested fix:** add optional "Kelas" select + "Wali Murid" picker to the create dialog, both marked optional with "lewati, isi nanti" affordance.

### MINOR — Skeleton persists ~5s before empty state on zero-data pages

/admin/leave, /admin/guardians, /admin/admissions, /admin/settings/users: header count updates to "0 ..." quickly, but the table skeleton keeps animating ~4-5s before the empty illustration renders. Looks stuck when data is known-zero.

**Why it matters:** adds perceived latency on every first visit to an empty module — exactly the pages a new tenant sees first.

**Suggested fix:** when the list query returns an empty array, replace skeleton with empty state in the same render pass (don't wait for a subsequent idle tick).

### MINOR — FEE-01 amounts not visible on /admin/fees Komponen tab

Columns shown: #, Komponen, Kategori, Aktif, action. No amount column. Amount is presumably on the "Struktur per Program" tab (not inspected to avoid mutation). For the JTBD "update fee component amount", Ibu Nur first has to realize the Komponen tab is not where amounts live.

**Suggested fix:** either surface amount(s) on Komponen tab (comma-separated per program), or rename tab labels to make the split obvious ("Definisi" vs "Tarif per Program").
