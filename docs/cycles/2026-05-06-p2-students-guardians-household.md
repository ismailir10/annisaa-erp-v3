### Assumptions

1. **Storage.objects RLS folded inline into migration 07** rather than separate `21_storage_rls/migration.sql`. Spec Â§18.2 caps cycles at 1 migration. Storage RLS is small enough (~20 SQL lines) to ride 07's transaction without coupling concerns dangerously. Alternative: defer to a tiny standalone cycle. Pick fold-inline by default; flip if /build review surfaces concern.
2. **NIS allocator API surface** lives at `lib/students/nis-allocator.ts` (not `lib/students/index.ts` barrel). Single named export `allocateNis`. Pure function â€” caller supplies prisma instance + transaction context.
3. **MIME verify reads first 12 bytes** (covers JPEG `FFD8FFE0/E1/E8`, PNG `89504E47 0D0A1A0A`, PDF `25504446 2D`, WEBP `52494646 ____ 57454250`). No `file-type` npm dep â€” Next.js 16 + edge-runtime + ESM-only file-type@v18+ has bundling friction; inline 5-row table is simpler + audit-friendly.
4. **Rate-limit storage backend = in-memory `Map<string, Bucket>`**. Single-region Vercel deploy assumed; counters lost on cold start (acceptable per spec Â§16). Multi-region/Redis migration is a documented p3+ task. Bucket eviction = on-write opportunistic (no setInterval â€” keeps the module stateless across worker reuse).
5. **`permission.ts` contract fix direction** = add a new `supabaseUserId` arg to `ResolveArgs`; keep `userId` for User.id queries (line 128). Line 149 employee query then uses `args.supabaseUserId`. Rationale: callers (session) already know both â€” explicit > magic. Single-arg-rename alternative loses information at call site. (Direction is locked here; T6 implements without re-evaluation.)
6. **NIS allocator advisory-lock granularity** = `(tenantId, academicYearId)` pair via `hashtext(tenantId || ':nis:' || academicYearId)`. Program is NOT in the lock key â€” sequence-bump is per-(tenant, year, program) triple in the table row, but contention is rare cross-program (different programs allocate different number ranges); locking at year level keeps deadlock surface small and matches v1 InvoiceNumberSequence pattern.
7. **Migration 07 RLS policies use the established `request.jwt.claims` JSON path** matching p1-identity-rls + p1-audit-timeline-files patterns. Tenant id read from `(current_setting('request.jwt.claims', true)::json->>'tenant_id')`.

10. **Pure-docs Playwright skip not applicable** â€” this cycle ships code; vitest gate runs; Playwright skip is justified because no UI lands (rebuild-window guard in CI still active).
11. **StudentIdentifier supports soft-delete.** `StudentIdentifier.deletedAt DateTime?` ships in migration 07. NIS-history requirement (spec Â§4.5 â€” "NIS reissues per cohort retain history, no overwrite") means a deleted PRIMARY row must NOT block a new PRIMARY assignment.
