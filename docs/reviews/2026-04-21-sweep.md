## Blockers (8)

These must land before the next `staging → main` cut.

1. **[DATA] `prisma/schema.prisma:308` — `EmailLog` has no `tenantId`**
   Cross-tenant data leakage risk. Add `tenantId String`, index, and filter all queries by tenantId.

2. **[DATA] `prisma/schema.prisma:37` — `User.tenantId` is optional (`String?`)**
   Breaks the tenant-scoping guarantee. Either make it required, or document the exact super-admin bootstrap path that requires nullability and gate everything else.

3. **[ERR] `app/admin/settings/salary-components/page.tsx:96` — toggle `fetch()` without `res.ok` check**
   Silent failure on a settings write path. Highest-risk silent swallow because it controls payroll config.

4. **[TEST] `lib/auth.ts` — no unit test**
   Auth-critical: `getSession()`, demo-mode fallback, role resolution all untested.

5. **[TEST] `lib/xendit/client.ts` — no unit test**
   Payment integration untested — can fail silently in production.

6. **[TEST] `lib/api/pagination.ts` — no unit test**
   Boundary cases (page=0, negative pageSize, invalid sortOrder) untested.

7. **[TEST] `lib/api/response.ts` — no unit test**
   `paginatedResponse()` contract untested — every list route depends on it.

8. **[CRUD] 7 entities have no admin UI at all** — see CRUD section for the list. Not a single hotspot but collectively a product blocker.

**Majors:**

| Severity | Location | Issue |

| Major | Cross-portal — assessment labels ✅ [cycle: 2026-04-21-nav-ia-hygiene] | Same feature surfaces under four different labels. Admin sidebar group + page header say **"Penilaian"**. Teacher bottom-nav says **"Nilai"**, page header says **"Nilai Siswa"**. Parent bottom-nav says **"Rapor"**, page header says **"Laporan Perkembangan"**. System model is `StudentAssessment`. Teacher-admins who switch roles see the same data under three different names. | Major | `components/teacher/bottom-nav.tsx:8-14` ✅ [cycle: 2026-04-21-nav-ia-hygiene] | Teacher bottom-nav went from 4 → 5 tabs (Beranda, Kehadiran, Kelas, Nilai, Gaji) inside a `max-w-md` (~448px) container. JTBD overlap: "Kelas" and "Nilai" both operate on class rosters. Options: (a) combine into a single "Kelas" tab with sub-nav, (b) move "Gaji" behind a profile/more menu since it's low-frequency, (c) accept 5 tabs and add overflow scroll at < 360px. |
| Major | `app/admin/assessments/scores/page.tsx` + `config/admin-nav.ts` ✅ [cycle: 2026-04-21-nav-ia-hygiene] | Assessment scoring detail uses `/admin/assessments/scores?id=<id>` query-param routing. | Major | `config/admin-nav.ts:99-115` — `getBreadcrumbs()` ✅ [cycle: 2026-04-21-nav-ia-hygiene] | Helper only handles depth-2 paths with a hardcoded suffix allowlist (`new`, `monthly`). Any deeper path (`/admin/employees/[id]/edit`, `/admin/assessments/templates/[id]`, `/admin/students/[id]/guardians/new`) collapses to "Detail", which is now visibly wrong on five admin routes. Fix: generalize — parse remaining segments, map `new` → "Tambah", `edit` → "Ubah", `[id]` → entity name via an async resolver or a short lookup table per entity. |
